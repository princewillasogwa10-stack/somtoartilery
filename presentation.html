Authentication Flow

Registration

  1. User submits name, email, password
  2. Server hashes password with bcrypt
  3. Creates JWT token (7-day expiry)
  4. Saves user to users.json
  5. Returns token to frontend

Protected Requests

  1. Frontend stores token in localStorage
  2. Sends as Authorization: Bearer <token>
  3. Server verifies with jwt.verify()
  4. Extracts user identity from payload
  5. Grants or denies access
// Auth middleware
function authMiddleware(req, res, next) {
  const payload = jwt.verify(token, JWT_SECRET);
  req.user = payload;
  next();
}

Inquiry Flow (Protected)

  1. User browses collection, adds artworks to inquiry bag
  2. Fills contact form (interest, selected works)
  3. Frontend sends POST /api/inquiry with JWT
  4. Server extracts name & email from the token
  5. Saves inquiry to submissions.json
  6. Sends email notification via Resend API
  7. Returns "Your interest has been sent."
⬇

User info comes from the JWT — no need to re-type name/email in the form.

Email Integration (Resend)

When an inquiry is submitted, the gallery owner gets an email:

From:    SOMTO ATELIER <onboarding@resend.dev>
To:      princewillasogwa10@gmail.com
Subject: New inquiry from [User Name]

Name:     User Name
Email:    user@example.com
Interest: Private viewing
Works:    Abstract Lady, Dreamscape

Submitted: 2026-06-24T21:40:14.556Z

Why Resend? Simple API, 100 free emails/day, no SMTP config. Just one API key.
